## 一、它是什么（30 秒理解）

把这个系统想象成客服坐席右手边的**资深前辈**：

- 坐席接电话、点按钮、说话术、做最终决定（坐席是决策者 / HITL）
- 系统在旁边帮你**查订单、判规则、起草话术、提醒禁区、留证据**
- 任何写操作（生成退货单）前，系统会弹出确认按钮 — 坐席手动确认才执行
- 通话结束后，退货记录自动归档到客户画像，下次同一客户再来能用到

**架构：Neuro-Symbolic Agent** — 规则引擎做骨架（决定对错），LLM 只做话术润色（决定好坏）。

│   ├── rules_engine.py      # 规则执行器

├── tools/                   # 原子工具

│   ├── orders.jsonl         # 20 条 Mock 订单
│   └── ecommerce.db         # SQLite 运行时数据库
├── frontend/

## 五、从 0 到 1 跑通一条退货

**终端模式 — 5 步走完全流程：**

```bash
# Step 1: 查看所有可用订单
python3 main.py list-orders

# Step 2: 启动退货会话（选 ORD001，模拟用户说"尺码不对太大"）
python3 main.py start --order ORD001 --text "尺码不对太大了想退货"
# → 记下输出的 session_id（如 a1b2c3d4）

# Step 3: 推进流程（点按钮）
python3 main.py event --session a1b2c3d4 --event CONTINUE       # 识别意图
python3 main.py event --session a1b2c3d4 --event USER_ACCEPTED   # 确认意图
python3 main.py event --session a1b2c3d4 --event CONTINUE       # 核资格
python3 main.py event --session a1b2c3d4 --event CONTINUE       # 算退款
python3 main.py event --session a1b2c3d4 --event CONFIRM_ACTION  # 确认执行

# Step 4: 查看结果
python3 main.py status --session a1b2c3d4
```

## 七、完整事件清单

| 事件 | 用途 | 适用节点 |
|------|------|---------|
| `CONTINUE` | 确认当前步骤，继续 | query_order, check_eligibility, calculate_refund |
| `USER_ACCEPTED` | 用户认可当前信息 | classify_intent |
| `USER_REJECTED` | 用户不认可 | classify_intent |
| `INTENT_SELECTED` | 切换意图（+payload: selected_intent） | classify_intent |
| `CONFIRM_ACTION` | 确认执行写操作（+idempotency_key） | execute_return |
| `TERMINATE` | 终止会话 | 任意节点 |

## 八、带到客户现场怎么讲（5 张图的故事）

**1️⃣ 架构总览** — 4 层架构（路由→编排→引擎→交互）

> "我们不是做了个 chatbot，是把退货业务建模成了可执行的规则引擎。
> 规则决定对错，LLM 只做话术润色。业务规则改了，改 YAML 就行。"

**2️⃣ 15 条规则** — 打开 `rules/return_rules.yaml`

> "6 条资格判定是 AND（全过才能退），7 条禁办是 OR（任一命中即拦截）。
> 每条规则都有 fail_action 和 fail_message，坐席不用记，系统自动判。"

**3️⃣ 5 步状态机** — 打开 `engine/state_machine.py`

> "查订单→识意图→核资格→算退款→执行。每个节点都明确说清楚：
> 进入时做什么、等什么事件、跳哪里、是不是写操作（需 HITL）。"

**4️⃣ 现场演示 3 个场景**
- H001 正常退货 + H002 质量问题 → 展示两条不同退款路径
- H004 内衣拒退 → 展示规则引擎拦截能力

**5️⃣ 审核轨迹**
> "每个事件、每次工具调用、每次坐席决策都有 trace。
> data/.state/audit.jsonl 包含全量审计日志。"

## 九、核心卖点（一句话版本）

> **"我们的方案让坐席从 '查 5 个系统、记 15 条规则、手打话术' 解放出来 —
> 系统把规则、查询、话术、记忆全部串起来，坐席只需要做 3 件事：
> 听用户说话、点确认按钮、用人的同理心。"**
